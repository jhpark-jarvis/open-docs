# 인프라 가이드

`infra/` 디렉터리는 배포, 운영, 클라우드 인프라 관련 작업을 위한 공간입니다.

OpenDocs는 나중에 ECS 기반 배포를 목표로 하므로, 이 디렉터리에는 실행 환경과 배포 절차를 코드로 관리하는 자료를 둡니다.

## 이 디렉터리의 역할

- 로컬 개발과 배포 환경을 분리합니다.
- 배포 구성은 코드로 관리하고, 손으로 맞추는 부분을 줄입니다.
- ECS 배포를 기준으로 컨테이너, 네트워크, 비밀값 구조를 준비합니다.

## 여기에 넣는 것

- Terraform 구성
- ECS task / service 정의
- VPC, subnet, security group 같은 네트워크 설정
- IAM role / policy
- 배포 자동화 관련 스크립트
- 운영용 설정 샘플

## 작업 원칙

- 인프라 코드는 서비스 코드와 분리합니다.
- 환경별 차이는 변수와 파일 구조로 구분합니다.
- 비밀값은 저장소에 직접 넣지 않습니다.
- 배포 가능한 최소 단위부터 단계적으로 확장합니다.
- 로컬 테스트용 설정과 운영용 설정은 명확히 분리합니다.

## 권장 구조

```text
infra/
├─ README.md
├─ terraform/
├─ ecs/
├─ networking/
└─ scripts/
```

## 앞으로 참고할 점

- ECS 배포를 염두에 두고 컨테이너 이미지를 설계합니다.
- 로컬 `docker-compose.yml`은 개발용이고, 실제 배포 정의는 여기서 따로 관리합니다.
- 운영 변경은 코드 리뷰와 함께 기록이 남도록 문서화합니다.
- 비밀값과 인프라 상태 파일은 저장소 정책에 맞게 관리합니다.
